NAME

Kill it.
It stays killed.

bgviewer is a free, open-source menu-bar kill-switch for macOS background services. Activity Monitor shows you the process. bgviewer shows you why it keeps coming back — and stops it at the launchd level, where resurrection is decided.

SYNOPSIS
curl -fsSL https://raw.githubusercontent.com/AroraShreshth/bgviewer/main/install.sh | bash

~700 KB · universal binary · macOS 13+ · zero dependencies · MIT

DESCRIPTION

Why your kill −9 doesn't work

When you kill a stray process on macOS, launchd often resurrects it instantly — a plist flag called KeepAlive. You are not fighting the process. You are fighting its supervisor.

without: $ kill -9 4021 → launchd: KeepAlive=true → respawned, pid 4022 → pid 4023 → ⟲ forever
with: bgviewer → Stop → launchctl bootout → verified: gone → stays gone.

Stubborn processes get TERM, then KILL. Every action is verified after it runs.

01

What it sees

Six unified sections. Everything running behind your back, in one dropdown.

~/Library/LaunchAgents Your launch agents — the things you (or an installer) told macOS to keep running.
/Library/LaunchAgents Machine-wide vendor agents: Zoom, Logitech, antivirus tools. The usual suspects.
brew services Every Homebrew service, with its real state.
TCP ports Every process holding a port — the dev server you forgot about at :5173.
resource hogs ≥15% CPU or ≥1 GB RAM with no port. The helper app quietly burning CPU for three days.
cron Your cron jobs. Read-only — bgviewer looks, doesn't touch.

Plus: opt-in notification the moment a new dev server starts listening — even while the dropdown is closed. Search by name, port, or command. Click any row for the full command line, plist, logs, and a one-click localhost:port. Auto-refreshes live.

02

The actions

BUTTON WHAT IT DOES DOES IT COME BACK?
Stop launchctl bootout — defeats KeepAlive. Stubborn processes get TERM, then KILL. Verified after it runs. no.
Pause / Resume Freeze the process without quitting it. Thaw when you want it back. when you say so.
Restart Clean stop, clean start. immediately — that's the point.
Disable Stops it now AND blocks auto-start forever. The plist is parked, never deleted. restored with one click.
(Apple system) Locked. Zero action buttons. n/a — you can't break your Mac.

03

Storage tools

Storage pane

Biggest files in Downloads, Desktop, Documents, Movies — with how long you've ignored them, and a free-space bar.

Disk Map

A clickable pie chart of any folder. Drill in wedge by wedge until you find the 40 GB surprise.

Dev Junk

Every node_modules, Python venv, Rust target, Pods, DerivedData across your projects, biggest first. The one place the app deletes — these rebuild on the next install.

It deletes exactly one category of thing, and only after checking twice. Dev Junk is guard-checked — node_modules only qualifies with a package.json beside it, guards are re-checked at delete time, two-step confirm. For personal files there is deliberately no delete button anywhere — reveal in Finder only.

INSTALL

Three ways in

1 — one-liner
curl -fsSL https://raw.githubusercontent.com/AroraShreshth/bgviewer/main/install.sh | bash
2 — Homebrew
brew tap arorashreshth/bgviewer https://github.com/AroraShreshth/bgviewer brew install --cask arorashreshth/bgviewer/bgviewer --no-quarantine
3 — manual: download the zip, drag to /Applications, then
xattr -d com.apple.quarantine /Applications/bgviewer.app

Honest caveat: not notarized yet, so macOS asks once — System Settings → Privacy & Security → Open Anyway. The one-liner handles it for you.
