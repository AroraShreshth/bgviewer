The kill-switch

Sees what Activity Monitor can't.
Stops what kill -9 can't.

macOS relaunches KeepAlive agents the instant you kill them. bgviewer unloads them at the launchd level — and can park them so they never come back.

⏹

Stop that sticks

launchctl bootout beats KeepAlive — no respawn. Stubborn processes get TERM, then KILL. Every action is verified after it runs.

⃠

Disable = parked, not deleted

Stops it now and blocks every future auto-start. The plist is parked, never deleted — one click brings it back.

🗂

Six sources, one list

Your launch agents, machine-wide vendor agents, Homebrew services, port listeners, resource hogs and cron — unified.

🔥

Install

In your menu bar in ten seconds.

One-liner RECOMMENDED

curl -fsSL https://raw.githubusercontent.com/AroraShreshth/bgviewer/main/install.sh | bash

Downloads the latest release, installs to /Applications, launches. Done.

Homebrew

brew tap arorashreshth/bgviewer \ https://github.com/AroraShreshth/bgviewer brew install --cask \ arorashreshth/bgviewer/bgviewer --no-quarantine

Checksum-pinned cask, straight from the repo.

Manual

Grab the zip from Releases, drag to /Applications, then:

xattr -d com.apple.quarantine \ /Applications/bgviewer.app

SHA-256 checksums ship with every release.

First-launch note: bgviewer isn't notarized yet (it's a free project — Apple's $99/yr comes later), so macOS asks once: approve via System Settings → Privacy & Security → Open Anyway, or use the one-liner, which handles it. After that, the built-in self-updater keeps you current automatically.

Why does the thing I killed keep coming back?

Because launchd is doing its job: agents with KeepAlive are relaunched the moment they die. Killing the process treats the symptom. bgviewer's Stop unloads the agent itself (launchctl bootout) — and Disable blocks it from ever auto-starting again.

Can it break my Mac?

No. Apple system processes are locked — shown for visibility, with no action buttons. User agents you disable are parked, never deleted, and restored with one click.

Does it phone home?

No telemetry of any kind. The only network access is checking GitHub for new releases (at most every 6 hours) and downloading an update when you — or auto-update, if you turn it on — say so. Updates are SHA-256 and signature verified.

Is deleting Dev Junk safe?

A folder only qualifies if it provably belongs to a project — node_modules needs a package.json beside it, .venv needs pyvenv.cfg inside, Rust target needs Cargo.toml. The guard re-runs at delete time and the button is two-step. They all rebuild on the next install.

What does it run on?

macOS 13 Ventura or newer, Apple Silicon and Intel (universal binary). It's a ~700 KB app with zero dependencies, built with a single swiftc command.
